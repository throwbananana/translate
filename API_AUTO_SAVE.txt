========================================
HOW IT WORKS
========================================

1. AUTO-SAVE ON EXIT
   When you close the program:
   - Configuration automatically saves
   - No need to click "Save"
   - Works even if you click the X button

2. AUTO-BACKUP SYSTEM
   Every time config is saved:
   - Creates timestamped backup
   - Keeps 3 most recent backups
   - Older backups auto-deleted
   - Location: config_backups/

3. AUTO-RESTORE ON FAILURE
   If config file is corrupted:
   - Automatically detects the problem
   - Restores from latest backup
   - Shows warning message
   - No data loss!

4. TEST CONNECTION
   In API configuration dialog:
   - Click "测试连接" button
   - Tests API key validity
   - Shows model response
   - Confirms connection works

5. SMART EXIT HANDLING
   If translation is running:
   - Asks "Are you sure?"
   - Option to cancel exit
   - Saves config anyway

========================================
BACKUP RETENTION POLICY
========================================

Maximum backups: 3

Retention logic:
  - Newest backup: Always kept
  - 2nd newest: Always kept
  - 3rd newest: Always kept
  - 4th+ oldest: Auto-deleted

Example timeline:
  10:00 - Backup 1 created
  11:00 - Backup 2 created
  12:00 - Backup 3 created
  13:00 - Backup 4 created (Backup 1 deleted)
  14:00 - Backup 5 created (Backup 2 deleted)

========================================
ERROR HANDLING
========================================

Scenario 1: Save Failure
  Problem: Cannot write config file
  Action: Shows error message
  Recovery: Check file permissions

Scenario 2: Backup Failure
  Problem: Cannot create backup
  Action: Logs error to console
  Recovery: Main save still succeeds

Scenario 3: Restore Failure
  Problem: No backups available
  Action: Shows warning
  Recovery: Uses default config

Scenario 4: Invalid Config
  Problem: JSON format error
  Action: Auto-restores from backup
  Recovery: Latest valid config loaded
