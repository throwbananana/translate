## 主要问题解决

您提出的问题："为什么导入最多只有2000-3000字，能否导入量大的文本呢？"

### 问题分析

经过测试，发现程序本身没有2000-3000字的硬性限制。实际问题是：
- tkinter的Text组件在显示大量文本时性能下降
- 导致界面卡顿，看起来像是"加载不完整"
- 虽然文本已完整加载到内存，但显示缓慢

### 解决方案

v1.1 引入了智能预览模式：

✅ **自动检测**：文件大小 > 10,000 字符时自动启用预览
✅ **流畅显示**：只显示前 10,000 字符，避免卡顿
✅ **完整翻译**：翻译时使用完整文本，不受显示限制
✅ **手动切换**：可点击按钮查看完整原文

## 技术实现

### 核心代码片段

```python
# 1. 检测文件大小
char_count = len(content)
is_large_file = char_count > self.preview_limit

# 2. 智能显示
if is_large_file and not self.show_full_text:
    # 预览模式
    preview = content[:10000]
    display_text = preview + "\n\n[预览模式提示]"
else:
    # 完整模式
    display_text = content

# 3. 翻译使用完整文本
segments = split_text(self.current_text)  # 完整文本
```

## 常见问题

### Q: 预览模式会影响翻译质量吗？

A: 不会！预览只影响显示，翻译使用完整文本。

### Q: 可以翻译多大的文件？

A: 理论上无限制。实际限制：
- 内存：能容纳文本即可
- API：某些API有token限制
- 时间：大文件需要更长时间

推荐范围：< 1,000,000 字符（约50万词）

### Q: 如果想一直看到完整文本怎么办？

A: 修改代码中的阈值：
```python
self.preview_limit = 999999999  # 设置极大值
```

### Q: 能否调整预览大小？

A: 可以，修改：
```python
self.preview_limit = 20000  # 改为20,000字符
```
